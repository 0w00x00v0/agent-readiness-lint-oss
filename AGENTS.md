# AGENTS.md

## Project Purpose

agent-readiness-lint is a local-only CLI tool that checks whether a repository is structured for effective AI coding agent workflows. It scans file structure, scores readiness 0-100, and generates actionable reports. No network calls, no APIs, no telemetry.

## Setup Commands

```bash
# Install in development mode with test dependencies
pip install -e '.[dev]'

# Or minimal install (no pytest/ruff)
pip install -e .
```

Requires Python 3.10 or later.

## Test Commands

```bash
# Run full test suite
pytest tests/ -v

# Run a specific test file
pytest tests/test_scoring.py -v

# Run with short traceback on failure
pytest tests/ -v --tb=short
```

All tests are deterministic, offline, and use temporary directories (no network, no real data).

## Lint Commands

```bash
# Ruff is configured in pyproject.toml (line length 100, target py310)
ruff check agent_readiness_lint/ tests/
```

CI (`.github/workflows/ci.yml`) runs both `ruff` and `pytest` on Python
3.10-3.12. It is offline apart from installing dependencies, uses no secrets,
and has no publish/release steps.

## File Boundaries

- **Source code:** `agent_readiness_lint/` - the installable Python package
  - `__main__.py` - CLI entry point and argument parsing
  - `checks/` - individual check modules (one per category)
  - `scoring.py` - score computation logic
  - `output.py` - report file generation
- **Tests:** `tests/` - pytest test suite with fixture-based repos
- **Documentation:** `docs/` - design docs, examples, and the public-candidate checklist
- **Example input:** `examples/sample-repo/` - committed fictional repo used to demonstrate the tool (generated reports are not committed)
- **CI:** `.github/workflows/` - GitHub Actions config (offline except dependency install)
- **Configuration:** `pyproject.toml` - project metadata, dependencies, tool config
- **Root docs:** `README.md`, `AGENTS.md`, `LICENSE`

## Generated Files

Do not commit the following (they are in .gitignore):

- `out/` - default output directory for generated reports
- `__pycache__/` - Python bytecode cache
- `*.egg-info/` - package metadata (generated by pip install -e)
- `.pytest_cache/` - pytest cache
- `.agents/` - agent task state (internal workflow tracking)
- `.venv/` - virtual environments

## Secrets and Private Data

This project uses no secrets, API keys, tokens, or credentials. It makes no network calls and accesses no external services.

Do not introduce:

- API keys or tokens
- Network calls
- Credential store access
- Browser profile access
- Environment variable reading (beyond standard Python paths)
- Real customer or personal data

## Non-Goals

- **Not a SaaS product.** This runs locally only.
- **Not a security scanner.** Secret risk checks are best-effort heuristics, not replacements for dedicated tools.
- **Not a bot or daemon.** No background processes, watchers, or scheduled tasks.
- **No telemetry.** No usage data collection of any kind.
- **No code execution.** The tool never runs code from the target repository.
- **No network access.** By design, the tool works fully offline.
- **No guarantees.** Scores are informational, not certifications.

## Conventions

- Python 3.10+ with type hints
- Line length: 100 characters (configured in pyproject.toml via ruff)
- Each check module is a standalone function returning a structured dictionary
- Test fixtures use pytest `tmp_path` for isolated temporary repositories
- Commit messages use type prefixes: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`
